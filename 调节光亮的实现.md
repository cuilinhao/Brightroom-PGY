


// 调节光亮的实现

调节图片的光亮，具体代码是在那实现的，怎么实现的，是调节layer吗？


用大白话说：**不是调 layer，是用 Apple 官方的 Core Image 滤镜直接处理像素数据**。

---

## 核心就一个地方

文件：`Sources/BrightroomEngine/Filter/FilterExposure.swift`

```25:46:Sources/BrightroomEngine/Filter/FilterExposure.swift
public struct FilterExposure : Filtering, Equatable, Codable, Sendable {
  
  public static let range: ParameterRange<Double, FilterExposure> = .init(min: -1.8, max: 1.8)
  
  public var value: Double = 0
  
  public init() {
    
  }
  
  public func apply(to image: CIImage, sourceImage: CIImage) -> CIImage {
    return
      image
        .applyingFilter(
          "CIExposureAdjust",
          parameters: [
            kCIInputEVKey: value as AnyObject
          ]
    )
  }
  
}
```

---

## 大白话解释

**调光亮 = 调 EV 曝光值，不是调 layer**

具体流程是这样的：

1. 你拖动滑块，产生一个 `-1.8 ~ 1.8` 之间的数字（比如拖到一半 = `0.9`）

2. 这个数字被传给 `CIExposureAdjust` —— 这是 Apple 的 **Core Image 内置滤镜**，直接对图片的每一个像素做数学运算，让像素变亮或变暗

3. 输入是一个 `CIImage`（原图），输出是一个新的 `CIImage`（处理后的图），原图不变

4. 最后用 **Metal GPU** 把处理结果画到屏幕上

---

## 和调 layer 的区别

| | 调 Layer（UIView.alpha / CALayer） | 这里的做法（Core Image） |
|---|---|---|
| 本质 | 改的是视图透明度，像蒙了层玻璃 | 真正改的是每个像素的亮度数据 |
| 效果 | 整体变透明，不是真正变亮 | 精确的曝光调整，和相机一样 |
| 导出 | 导出图片拿不到效果 | 导出的图片像素已经变了 |
| 性能 | GPU合成，很快 | 也是GPU（Metal），同样很快 |

---

## `kCIInputEVKey` 是什么

EV = Exposure Value，就是摄影里的「曝光档数」。

- `0` = 不变
- `+1` = 亮一档（每个像素亮度 × 2）
- `-1` = 暗一档（每个像素亮度 ÷ 2）

这是行业标准的曝光调节算法，Apple 直接封装好了，一行 `applyingFilter("CIExposureAdjust", ...)` 就搞定。
